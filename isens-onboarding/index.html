INTRO · 온보딩의 정의

온보딩은 교육이 아니라,
셋업입니다.

교육은 끝나면 기억만 남습니다. 우리는 아이센스의 실제 데이터 위에 택소노미·도입 가이드·QA·AI 셋업을 얹어, 온보딩이 끝난 뒤에도 계속 쓰이는 물건을 남깁니다.

INTRO · 아이센스의 조건

아이센스에만
있는 세 가지 조건

표준 온보딩은 앱 1개, 백지 상태, 일반 데이터를 전제합니다. 아이센스는 셋 다 다릅니다.

조건 01

앱이 2개다

신규 앱과 기존 운영 앱.
도입 가이드와 QA를
앱별로 두 벌 돌립니다.

조건 02

데이터가 이미 있다

신규 앱은 택소노미 설계 완료,
기존 앱은 설계 자체가 없음.
백지가 아닙니다.

조건 03

민감정보를 다룬다

의료 데이터 · EU/영국 DPA.
개인정보·민감정보를 해싱·가명처리한 뒤
분석하는 설계가 필요합니다.

그래서

표준 스콥을 그대로 쓰지 않고, 아이센스 조건에 맞게 재배치합니다.

MIXPANEL온보딩제안
제안 ① · 구조

단계마다 남는 것이 있습니다

PHASE I

플래닝 Planning

손실 없이 넘어오기

  • 킥오프 · 온보딩 로드맵 수립
  • 신규 앱 — 기존 택소노미 리뷰 · 정합성 진단
  • 기존 앱 — MVP 택소노미 우선순위 설계안 리뷰
PHASE II

구현 Implementation

처음부터 맞게 심기

  • SDK 구현 가이드 — 개발자 테크니컬 가이드
  • QA 가이드 + QA 지원 — 수집 무결성 확인·교정
  • 구현 중 이슈 실시간 대응 (슬랙)
PHASE III

AI 활성화 Enablement

묻는 조직으로

  • 핵심 리포트 활용 — 인사이트·퍼널·플로우·리텐션
  • 조직 분석 질문 도출 · BQ 리포트 작성
    BQ = Business Question, 부서별 비즈니스 질문
  • AI Agent · MCP 셋업 — 자동화·알림·역할별 활용
선긋기

온보딩은 가이드입니다. 직접적인 설계·분석·구축 대행은 포함되지 않습니다.

  • MVP 택소노미 우선순위 설계안
    기존 앱 — 촉박한 일정에서 뭘 먼저 심을지
II · 구현
  • SDK 구현 가이드 (앱별)
    개발팀이 들고 바로 작업
  • QA 체크리스트 + 검증 결과
    데이터를 믿을 수 있는 근거
III · AI 활성화
  • 아이센스 BQ 세트
    부서별 비즈니스 질문(Business Question) → 리포트
  • 핵심 리포트 템플릿 (실데이터 4종)
  • AI Agent 컨텍스트 셋업
    물으면 답이 나오는 상태
  • MCP 연결 가이드
    역할별 — PM·기획·분석·개발

신규 앱

이식 · 정합화

외주 택소노미가 이미 있습니다.
새로 설계하지 않고 믹스패널에서 실제로 도는지 검증합니다.
