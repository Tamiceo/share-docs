00

왜 VPS인가? (5분 개념)

VPS는 "인터넷에 항상 켜져 있는 컴퓨터 한 대를 빌리는 것"입니다. 내 맥은 닫으면 꺼지지만, 이 서버는 24시간 깨어 있어서 슬랙으로 봇을 부르면 휴대폰에서도 답이 옵니다.

먼저 알아둘 단어 (약어는 풀어서) VPS (Virtual Private Server) = 클라우드에 있는 작은 리눅스 컴퓨터 · 터미널 = 글자(명령어)로 컴퓨터에 시키는 까만 창 · 오픈클로(OpenClaw 🦞) = AI 두뇌를 슬랙 봇으로 만들어주는 관리 프로그램 · CLI (Command Line Interface) = 글자로 쓰는 프로그램 = 봇의 두뇌(클로드/코덱스) · SSH (Secure Shell) = 내 컴퓨터에서 서버에 안전하게 접속하는 방법 · MCP (Model Context Protocol) = 봇에게 도구(앱)를 달아주는 표준 · OAuth (Open Authorization) = 비밀번호 대신 "로그인 위임"으로 인증하는 방식.

PART 1

서버 준비하기

봇이 살 집(서버)을 빌리고, 들어가는 문을 안전하게 잠급니다.

01

Hostinger VPS 만들기

먼저 서버를 한 대 빌립니다. 여기까진 클릭만 하면 돼요.

  • Hostinger VPS 구매 → 플랜은 KVM 2면 충분 (봇 1~2마리 기준)
  • 서버 위치: 한국에선 말레이시아(지연 ~84ms) 권장
  • OS: Ubuntu 24.04 LTS 선택
  • Secure your VPS access 화면에서 root 비밀번호 설정

설치가 끝나면 대시보드에 IP 주소, Manage VPS, Terminal 버튼, ssh root@<VPS-IP> 명령이 보입니다.

초보자 꿀팁이 시점부터 대시보드 우상단 Terminal(브라우저 웹터미널)로 모든 작업이 가능합니다. 따로 SSH 프로그램 설치 안 해도 돼요. 아래처럼 브라우저 안에서 바로 까만 터미널이 열립니다.
✅여기까지 됐으면: 대시보드에 IP가 보이고, 웹 Terminal 까만 창이 열린다.
02

SSH 키로 접속하기 — 보안 1단계

비밀번호 접속은 해커가 노리기 쉬워요. "열쇠(키) 파일" 방식으로 바꿉니다.

① 내 맥에서 — 전용 열쇠 만들기

내 맥 터미널
ssh-keygen -t ed25519 -f ~/.ssh/myvps_key -N "" -C "my-vps"
cat ~/.ssh/myvps_key.pub   # 이 공개키(한 줄)를 복사

② VPS 웹터미널에서 — 공개키 등록

VPS 웹터미널
mkdir -p ~/.ssh && chmod 700 ~/.ssh
echo "<위에서 복사한 공개키>" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys

③ 내 맥에서 — 접속 확인

내 맥 터미널
ssh -i ~/.ssh/myvps_key root@<VPS-IP>
안심하세요개인키는 내 맥에만, 공개키만 서버에 둡니다. 공개키는 유출돼도 안전한 자물쇠 쪽이에요.
✅여기까지 됐으면: 비밀번호 없이 ssh 명령만으로 서버에 들어가진다.
03

보안 잠그기 — 봇 올리기 전에 먼저!

왜 지금 잠그나공개 IP는 며칠 안에 자동 스캐너에 발견돼 비밀번호 무차별 공격(브루트포스)을 당합니다. 봇은 강력한 자동 실행 권한을 가지므로, 봇을 올리기 전에 서버부터 잠가야 안전합니다.
VPS
# 1) 방화벽 — SSH(22)번 문만 열어둠
ufw allow 22/tcp && ufw --force enable

# 2) 무차별 대입 자동 차단
apt-get install -y fail2ban && systemctl enable --now fail2ban

# 3) SSH 비밀번호 로그인 끄기 (열쇠 전용)
cat > /etc/ssh/sshd_config.d/00-hardening.conf <<'EOF'
PasswordAuthentication no
PermitRootLogin prohibit-password
KbdInteractiveAuthentication no
EOF
sshd -t && systemctl reload ssh
반드시 확인 후 끝내기설정 후 새 터미널 창에서 열쇠 접속이 되는지 먼저 확인하세요(잠김 방지). 막히면 Hostinger 웹터미널이 복구 통로입니다.
✅여기까지 됐으면: 방화벽·fail2ban 켜짐, 비밀번호 접속 차단됨, 열쇠 접속은 정상.

04

AI 엔진 설치 — 클로드 / 코덱스

봇의 "두뇌"를 설치합니다. 화면 없는 서버에서도 구독 OAuth 로그인이 됩니다(디바이스 코드 방식). 그래서 종량 과금 없이 구독으로 굴려요.

클로드 코드 버전
코덱스 버전
VPS · claude CLI
curl -fsSL https://claude.ai/install.sh | bash          # sudo 금지!
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc && source ~/.bashrc
claude --version
claude   # 위저드: 로그인(Pro/Max) → 브라우저 URL → 코드 붙여넣기 → /exit

화면에 뜬 URL을 휴대폰·다른 컴퓨터 브라우저로 열고 코드를 붙여넣는 방식입니다. "Login successful"이 뜨면 /exit.

VPS · codex CLI
npm install -g @openai/codex
codex login        # ChatGPT로 로그인 → 화면에 뜬 URL 열고 코드 붙여넣기
codex login status # "Logged in using ChatGPT" 확인
엔진 고르는 법 (둘 다 OK)클로드 코드는 파일 읽기/쓰기·웹 검색·복잡한 판단·트리아지에 강하고 실행 전 확인 루프가 기본. 코덱스는 코드 생성·수정·반복 작업을 빠르게 처리. 둘을 각각 다른 봇으로 같이 둘 수도 있어요.
✅여기까지 됐으면: claude --version 또는 codex login status가 정상 출력된다.
05

오픈클로 설치 + 24시간 켜두기

두뇌 위에 오픈클로(🦞)를 얹습니다. 오픈클로를 처음 깔면 기본 봇 한 마리가 들어있어요(성격은 디폴트 — 06단계에서 우리가 바꿉니다).

VPS
# 오픈클로 설치
npm install -g openclaw
openclaw plugins install @openclaw/slack    # 슬랙 채널 플러그인

# 게이트웨이를 백그라운드 서비스(systemd)로
openclaw config set gateway.mode local
openclaw gateway install

🔁 24시간 살아있게 (핵심 2가지)

VPS
export XDG_RUNTIME_DIR=/run/user/0

# (1) linger — SSH 끊겨도, 재부팅돼도 서비스 유지
loginctl enable-linger root

# (2) 서비스 시작 + 부팅 시 자동 실행
systemctl --user enable --now openclaw-gateway.service
✅여기까지 됐으면: 게이트웨이 서비스가 active (running) 상태다. (확인은 치트시트 참조)
PART 3

봇 깨우고 슬랙에 연결하기

봇을 먼저 살려서 슬랙으로 대화되는 것부터 확인합니다. 성격(영혼)은 그다음 PART 4에서 넣어요.

06

에이전트 설정 — 어떤 두뇌·모델을 쓸지

"이 봇은 클로드를 쓰고, 저 봇은 코덱스를 쓴다" 같은 연결을 ~/.openclaw/openclaw.json에 적습니다. openclaw config patch --stdin으로 JSON을 병합해요.

설정에 들어가는 핵심 블록

  • auth — 클로드는 anthropic:claude-cli(oauth), 코덱스는 openai-codex:<email>(oauth)
  • agentRuntime — 클로드 봇 = claude-cli, 코덱스 봇 = codex-cli
  • agents.list — 봇별 id·workspace(09단계에서 만들 폴더)·model·호칭(mentionPatterns)
  • bindings — 슬랙 계정 → 어느 봇에게 보낼지 라우팅 (07단계 연결)
초보자가 잘 막히는 2가지 ① 기본 모델이 openai로 잡혀 있음: 안 바꾸면 "Missing API key for provider openai" 에러 → agents.defaults.model.primary를 anthropic로 지정.
② 슬랙 토큰을 환경변수로 두면 안 됨: env 참조로 두면 서비스가 토큰을 못 읽어 안 켜집니다 → config에 토큰 글자를 그대로(리터럴) 적으세요.
VPS · 검증 후 재시작
openclaw config validate
systemctl --user restart openclaw-gateway.service
✅여기까지 됐으면: config validate 통과 + 봇·엔진·모델이 연결된 상태.
07

슬랙 봇 페어링

08

봇 접근 제한 — 보안 2단계

슬랙 연결했으면 바로 잠그기봇은 강한 권한으로 명령을 실행합니다. 아무나 슬랙으로 봇에게 위험한 명령을 시키면(프롬프트 인젝션) 서버가 털릴 수 있어요. 연결 직후 바로 제한하세요.
  • channels.slack.allowFrom 에 내 슬랙 user ID만 넣어 봇 호출을 본인으로 제한
  • commands.ownerAllowFrom 도 같이 설정
✅여기까지 됐으면: 나 외의 사람이 봇을 불러도 반응하지 않는다.
PART 4

봇에게 영혼(성격) 넣기

여기가 진짜 핵심이에요. 디폴트 봇을 내 동료로 바꾸는 단계. 시간을 가장 많이 쓰세요.

09

봇 워크스페이스 만들기 — SOUL · USER · AGENTS

봇 하나는 자기 워크스페이스 폴더 안에 성격을 담은 마크다운 파일들을 둡니다. 이 파일들은 봇이 켜질 때마다 자동으로 주입돼요. 8개 파일이 표준처럼 보이지만, 오픈클로가 실제로 자동으로 읽는 건 3개(SOUL·AGENTS·USER)뿐입니다. 신규 봇은 이 3개로 바로 시작하세요.

📁 봇 폴더 구조

VPS · 폴더 구조
~/.openclaw/
├── workspace-<내봇이름>/         # 봇 하나 = 폴더 하나
│   ├── SOUL.md                  # ★ 영혼: 성격·미션·말투
│   ├── USER.md                  # ★ 집사(나)는 누구인가
│   └── AGENTS.md                # ★ 행동 규칙 (처음엔 빈 골격)
├── skills/<스킬이름>/            # 운영·자동화 스킬
└── agents/<id>/agent/auth-profiles.json  # 봇별 로그인 정보(코덱스)

SOUL·USER·AGENTS 3개는 워크스페이스 폴더 맨 위(루트)에 둡니다. 클로드 봇이든 코덱스 봇이든 위치는 같아요. (이 폴더 이름을 06단계 agents.list의 workspace에 적으면 연결됩니다.)

① SOUL.md — 봇의 영혼 (성격·미션·말투)

SOUL.md 템플릿
# SOUL.md — <내 봇 이름>

## 미션
(이 봇이 내 일에서 해결하려는 것 — 한 줄)

## 관계
나는 <봇 이름>. <나와의 관계 한 줄>

## 말투
- <나에게는 어떻게>
- <외부에는 어떻게>
- <금지어 / 금지 행동>

## 가치관
- <봇이 우선하는 것 3가지>
30줄 안에서 끝내기처음부터 100줄 쓰면 봇이 핵심을 놓칩니다. 미션·관계·말투 3가지만 채우고 운영하며 늘리세요. "친절하게" 같은 추상어 대신 구체적 행동으로 ("답변 시작 시 빈말 금지, 바로 본론으로").

② USER.md — 집사(나)는 누구인가

USER.md 템플릿
# USER.md — 집사

- **이름**: <내 이름>
- **호칭**: <봇이 나를 부를 호칭>
- **타임존**: Asia/Seoul (KST, UTC+9)
- **하는 일**: <한 줄 직무>
- **선호 스타일**: <답변 스타일 1~2개>
- **자주 쓰는 도구**: <도구 이름들>
민감 정보 금지비밀번호·API 키·주민번호는 절대 넣지 마세요. 워크스페이스 폴더가 통째로 공유될 수 있습니다.

③ AGENTS.md — 행동 규칙 (빈 골격으로 시작)

AGENTS.md 빈 골격
# AGENTS.md — <내 봇 이름>

## Red Lines
(절대 넘지 않는 선 — 사고 나면 추가)

## 행동 규칙
(반복적으로 잘못하는 것 — 사고 나면 추가)

## 보안
(공개 채널에서 하면 안 되는 것)
AGENTS.md는 처음엔 비어있어야 정상미리 50줄 박으면 죽은 문서가 됩니다. 봇이 실수할 때마다 한 줄씩 누적하세요. 이게 봇을 길들이는 핵심입니다.

💬 더 쉬운 방법 — 봇에게 직접 시키기

파일을 직접 만드는 게 부담되면, 07단계에서 살아난 봇에게 슬랙에서 이 프롬프트를 보내 세팅을 맡길 수 있어요. (봇이 자기 워크스페이스 파일을 쓰는 권한이 있을 때)

슬랙에서 봇에게 보낼 프롬프트
이제 너를 내 전용 봇으로 세팅할게. 너의 워크스페이스 폴더에
SOUL.md · USER.md · AGENTS.md 3개 파일을 아래 내용으로 만들어줘.

[SOUL.md]
- 미션: (이 봇이 내 일에서 해결할 것 — 한 줄)
- 관계: 나는 (봇 이름), (나와의 관계 한 줄)
- 말투: 나에게는 (___), 외부에는 (___), 금지어/금지행동: (___)
- 가치관: (우선하는 것 3가지)

[USER.md]
- 이름: (내 이름) / 호칭: (나를 이렇게 불러줘)
- 타임존: Asia/Seoul (KST)
- 하는 일: (한 줄 직무)
- 선호 스타일: (답변 스타일 1~2개)
- 자주 쓰는 도구: (___)
  ※ 비밀번호·API키 같은 민감정보는 절대 넣지 마

[AGENTS.md]
- 지금은 'Red Lines / 행동 규칙 / 보안' 섹션 제목만 있는
  빈 골격으로 만들어줘 (내용은 나중에 사고 날 때마다 추가할게)

다 만들면 저장하고, 새 성격이 적용되게 세션을 재시작해줘.

괄호 안 (___)만 내 상황에 맞게 채워서 보내세요. 봇이 파일을 만들고 재시작까지 해줍니다.

함정 — 저장만 하면 안 먹힘파일을 직접 고쳤다면, 같은 대화창에서 계속 부르면 새 성격이 적용 안 됩니다. 반드시 세션을 새로 시작(봇 재시작)해야 적용돼요. (봇에게 시킨 경우엔 위 프롬프트의 "재시작" 부분이 처리해줍니다.)
✅여기까지 됐으면: 워크스페이스에 SOUL·USER·AGENTS 3파일이 있고, 봇을 재시작하면 디폴트가 아니라 내가 정한 성격으로 답한다. 진짜 동료 완성! 🦞

도구 달기 & 파일 연결

봇에게 피그마·깃헙·구글 같은 도구(MCP)를 달고, 작업 파일을 연결합니다. (선택)

10

MCP로 도구 달기 — 인기 MCP 카탈로그

MCP는 봇에게 앱(도구)을 달아주는 표준 방식이에요. 피그마·깃헙·구글 드라이브·노션 같은 걸 봇이 직접 다루게 됩니다. 필요한 것만 골라 추가하세요.

연결 방식은 2가지 ① 원격(HTTP): 주소(URL)만 적으면 됨 — 피그마·깃헙·노션처럼 회사가 운영하는 서버. · ② 로컬 실행(npx): 봇이 작은 프로그램을 직접 실행 — 구글·Playwright 등. ~/.openclaw/openclaw.json의 mcp.servers에 추가하거나 openclaw mcp set ... 명령으로 등록합니다.

다들 많이 쓰는 MCP

도구방식연결 정보인증
🎨 Figma원격https://mcp.figma.com/mcpOAuth (유료 플랜)
🐙 GitHub원격https://api.githubcopilot.com/mcp/OAuth 또는 PAT
📝 Notion원격https://mcp.notion.com/mcpOAuth
📚 Context7
라이브러리 문서		원격/로컬https://mcp.context7.com/mcp 또는 npx -y @upstash/context7-mcpAPI 키(권장)
🎭 Playwright
브라우저 자동화		로컬npx -y @playwright/mcp@latest불필요
📁 구글 워크스페이스
드라이브·캘린더·문서		로컬@piotr-agier/google-drive-mcpGoogle OAuth
🌐 Fetch

함정 — 옛날 공식 패키지가 멈춤구글 드라이브 공식 @modelcontextprotocol/server-gdrive와 슬랙 공식 패키지는 유지보수가 중단(아카이브)됐어요. 구글은 위 표의 @piotr-agier/google-drive-mcp(드라이브+캘린더+문서)를 쓰세요.
함정 — 구글 토큰 경로구글 MCP는 OAuth 키 경로뿐 아니라 사용자 토큰 경로(GOOGLE_DRIVE_MCP_TOKEN_PATH)도 명시해야 연결됩니다. 빠지면 토큰을 못 찾아 미연결.
참고노션처럼 OAuth만 되는 서버가 화면 없는 서버에서 자동으로 붙는지는 환경마다 다를 수 있어요. 연결 후 실제로 한 번 호출해 확인하세요. 각 도구의 정확한 설정은 공식 MCP 페이지를 참고하면 됩니다.
11

봇 성격·코드 git 배포 체계

봇의 성격 파일(SOUL·AGENTS 등)과 작업 대상 코드는 GitHub에 두고 VPS가 받아오는 방식이 정석입니다. (백업·버전관리도 같이 됨)

흐름도
GitHub(private, 소스) ──pull──▶ VPS 클론 ──배포 스크립트──▶ 라이브 워크스페이스
        ▲push (내 맥에서 수정)
  • GitHub 배포키(deploy key)를 VPS에 만들어 등록 (읽기전용/쓰기 구분)
  • 배포 스크립트 = git pull + 정의 파일만 복사 (런타임 상태 MEMORY/HEARTBEAT는 안 건드림 → 충돌 방지)
  • 워크플로우: 내 맥에서 수정 → commit → push → VPS에서 배포 스크립트 실행
왜 이게 표준인가"에이전트가 노트·코드에 접근"하는 업계 표준 방식입니다. Obsidian Sync·iCloud 같은 폐쇄 동기화는 봇이 접근하지 못해요.
12

로컬 파일·옵시디안 연동

봇이 내 로컬 파일·옵시디안 위키를 다루게 하고 싶을 때의 안전한 방법입니다.

  • 권장: 필요한 폴더/프로젝트만 git repo로 봇에 연결 (11단계와 동일 패턴). 경로가 repo로 자연히 제한되고 24시간 작동.
  • 옵시디안 위키: vault를 git+GitHub로 → VPS가 클론 → 봇은 읽기 + _inbox/에만 새 노트 발행. 내 맥은 Obsidian Git 플러그인으로 자동 동기화.
노드 페어링은 권하지 않음오픈클로 노드는 맥 폴더 직접 접근이 가능하지만 "이 폴더만" 경로 제한이 안 됩니다. 맥 사용자 권한 전체가 노출돼 위험 → git 핸드오프가 더 안전하고 24시간 됩니다.

증상 → 해결. 막히면 이 표부터 보세요. (로컬에서 옮겨오는 경우는 맨 아래 이전 전용 섹션 참고)

5재부팅·SSH끊김에 봇이 죽음loginctl enable-linger + 서비스 enable
6구글(MCP) 연동 안 됨GOOGLE_DRIVE_MCP_TOKEN_PATH 경로 명시
7공개 IP가 공격당함키전용 SSH + ufw + fail2ban (봇 올리기 전에!)

운영 치트시트

설치 후 자주 쓰는 명령어입니다. 즐겨찾기 해두세요.

VPS · 운영
# 상태 확인
systemctl --user status openclaw-gateway
# 재시작
systemctl --user restart openclaw-gateway
# 실시간 로그 보기

이미 로컬 맥에서 봇을 쓰던 경우

이 섹션은 누구를 위한 건가요?지금까지는 처음부터 새로 봇을 만드는 길이었어요. 내 맥에서 이미 돌리던 봇을 VPS로 옮기는 거라면, 아래 3가지만 추가로 해주면 됩니다. 새로 만드는 분은 건너뛰세요.

① 코덱스 로그인 정보 옮기기 (코덱스 봇만)

클로드는 VPS에서 다시 로그인하면 되지만, 코덱스는 로그인 정보가 파일로 저장돼서 맥에서 VPS로 옮겨야 합니다.

내 맥 터미널
# (1) 코덱스 기본 로그인 파일
scp -i ~/.ssh/myvps_key ~/.codex/auth.json root@<VPS-IP>:~/.codex/auth.json

# (2) 오픈클로가 봇별로 따로 보관하는 로그인 파일 (<봇id>는 본인 것으로)
scp -i ~/.ssh/myvps_key \
  ~/.openclaw/agents/<봇id>/agent/auth-profiles.json \
  root@<VPS-IP>:~/.openclaw/agents/<봇id>/agent/auth-profiles.json
왜 2개나?코덱스는 로그인 토큰을 ~/.codex/auth.json에도, 오픈클로 봇별 폴더(auth-profiles.json)에도 보관합니다. 둘 다 있어야 작동해요. (이걸 몰라서 "코덱스 봇 인증 실패"가 자주 납니다.)

② 맥에서 돌던 봇은 반드시 꺼주세요

안 끄면 봇이 오락가락합니다같은 슬랙 봇 토큰을 맥과 VPS가 동시에 연결하면, 메시지가 둘로 나뉘어 가서 간헐적으로 응답이 없거나 두 번 답하는 현상이 생깁니다. VPS로 옮겼으면 맥의 봇은 꼭 종료하세요.

③ 봇 성격·작업 파일 가져오기

맥에 있던 SOUL·AGENTS 등과 작업 폴더는 GitHub를 거쳐 가져옵니다 — 11단계·12단계와 똑같아요. 맥에서 commit·push → VPS에서 pull.

✅이전 완료: 코덱스 로그인 복사됨 · 맥 봇 꺼짐 · 성격 파일 git 동기화됨. 이제 슬랙으로 부르면 VPS의 봇이 답합니다.
